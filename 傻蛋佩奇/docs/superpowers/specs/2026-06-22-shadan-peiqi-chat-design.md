# 傻蛋佩奇聊天 AI 设计

## 目标

制作一个纯本地可打开的聊天网页，名字叫“傻蛋佩奇”。它不接入 API，也不依赖网络服务。用户可以在浏览器中输入消息，傻蛋佩奇用呆萌可爱的本地规则回复，并且每一次机器人回复前都固定带上口头禅：`~哼哼~`。

## 范围

- 新建 `傻蛋佩奇/index.html`。
- 页面可通过双击 HTML 文件或浏览器直接打开使用。
- 不需要构建工具、后端服务、API key、外部字体或外部图片。
- 机器人回复由本地 JavaScript 规则生成，不宣称是真正联网大模型。

## 界面

页面采用可爱但清爽的聊天应用布局：

- 顶部显示圆形“傻蛋佩奇”头像、名称和在线状态。
- 中间显示聊天气泡，区分用户和傻蛋佩奇。
- 底部提供输入框、发送按钮和清空按钮。
- 支持移动端和桌面端自适应。
- 颜色以粉色为主，但搭配浅蓝、白色和深色文本，避免单一色块。

## 交互

- 用户点击发送或按 Enter 发送消息。
- 空消息不发送，并在输入框中给出轻量提示。
- 发送后立即显示用户气泡，再显示一个短暂的“傻蛋佩奇正在想”的状态。
- 机器人回复必须经过统一的 `withCatchphrase()` 包装函数，确保前缀始终是 `~哼哼~`。
- 清空按钮会恢复初始欢迎语。

## 本地回复规则

回复逻辑按关键词选择语气：

- 问候：回应打招呼。
- 名字/身份：介绍自己是傻蛋佩奇。
- 心情/难过：给可爱安慰。
- 学习/作业：鼓励用户慢慢来。
- 游戏/玩：表现出兴奋和陪玩感。
- 夸奖：害羞回应。
- 默认：用呆萌方式接住用户的话题。

所有回复都短句化，保持可爱、轻松、儿童友好。

## 验证

- 打开 `傻蛋佩奇/index.html` 后能看到完整聊天界面。
- 发送任意消息后机器人回复出现。
- 多种关键词和默认回复都以 `~哼哼~` 开头。
- Enter 发送、点击发送、清空聊天都能工作。
- 页面在窄屏下没有明显文字重叠或按钮溢出。
