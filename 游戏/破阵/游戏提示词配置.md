# 《忍影试炼》游戏提示词配置

## 使用原则

这套提示词用于制作一款“原创忍者题材”的 2D 动作游戏。可以参考热血忍者动画的节奏、战斗机制和视觉语言，但不要直接使用现成作品里的角色、村名、标志、服装或剧情。

当前项目核心设定：

- 游戏名：《忍影试炼》
- 类型：2D 动作 / 忍者格斗 / 闯关 + 对战
- 核心资源：体力、查克拉、替身术冷却
- 当前操作：WASD 移动，鼠标瞄准，左键普攻，Q/E/R/Z 忍术，C 蓄查克拉，Space 冲刺
- 当前角色：岚 · 均衡流、绯月 · 雷影流、铁心 · 岩铠流
- 扩展角色池：见 [角色设置.md](/Users/liuyueqin/Desktop/皓宇游戏/游戏/破阵/角色设置.md) 与 [角色设置.json](/Users/liuyueqin/Desktop/皓宇游戏/游戏/破阵/角色设置.json)

## 第一阶段：核心概念与 GDD

### 提示词 1：定义游戏类型与核心玩法

```text
角色：
你是一位拥有 20 年经验的资深动作游戏策划，擅长 2D 横版动作、格斗游戏与原创忍者题材。

任务：
请为一款名为《忍影试炼》的 2D 横版动作游戏设计核心玩法文档。

背景：
游戏设定在原创忍者世界。玩家需要体验“体术、忍术、幻术/扰乱”的战斗层次，但不要使用任何已有动漫作品的角色、地点、组织或标志。

要求包含：
1. 核心循环：移动、普通攻击、冲刺、跳跃、空中攻击、落地取消的连贯性。
2. 查克拉系统：限制忍术使用的能量条，包含自然恢复、攻击恢复、长按 C 蓄力恢复。
3. 战斗维度：近战体术/苦无、远程手里剑/忍术、位移追击、破防与硬直。
4. 替身术系统：受击瞬间可消耗查克拉触发，获得 0.5 秒无敌，在原地留下木桩替身，并瞬移到敌人背后或侧后方。
5. 对战模式：玩家选择 1 名角色，AI 随机选择不同角色，双方先击倒对方者获胜。
6. 闯关模式：多波敌人 + 关底 Boss。
7. 敌人 AI：设计 3 种基础敌人：下忍型、精英中忍型、傀儡型，描述移动、攻击、忍术、受击反应。
8. 角色差异：为岚、绯月、铁心分别定义血量、速度、攻击节奏和技能定位。

输出格式：
请用结构化 Markdown 输出，包含“核心体验目标”“玩家操作”“战斗系统”“敌人设计”“模式设计”“可扩展方向”。
```

### 提示词 2：技能树与成长系统

```text
上下文：
基于《忍影试炼》的原创忍者动作游戏设定。

任务：
请设计一个忍术学习与成长系统。

细节：
玩家通过战斗获得“忍术卷轴”，用来解锁或强化技能。技能分为三类：
- 体术：近战、连招、浮空、终结技。
- 忍术：远程、范围、控制、爆发。
- 扰乱术：替身、分身、位移、幻影诱导。

要求：
1. 设计至少 5 个基础技能。
2. 每个技能包含：名称、类别、伤害类型、查克拉消耗、冷却时间、解锁条件、升级效果、视觉特效。
3. 技能名称必须原创，但可以有“火、风、雷、土、影、分身”等忍者题材元素。
4. 给出适合岚、绯月、铁心的专属升级路线。

示例方向：
- 火轮弹：向前发射火球，爆炸造成范围伤害。
- 影步分身：短距离位移并留下可攻击的残影。
- 岩壁守势：生成短暂防御壁，抵挡远程攻击。
- 星雨手里剑：扇形投掷多枚手里剑。
- 多重残影阵：奥义技能，召唤多个残影连续斩击。

输出格式：
请用 Markdown 表格输出技能列表，并补充一段成长曲线说明。
```

## 第二阶段：代码逻辑实现

### 提示词 3：玩家状态机

```text
任务：
请用 TypeScript 为《忍影试炼》编写一个玩家状态管理类。

功能需求：
状态包括：
- Idle 待机
- Run 奔跑
- Jump 跳跃
- Fall 下落
- Attack 攻击
- Cast 施法
- Dash 冲刺
- Charge 蓄查克拉
- Stun 硬直
- Substitution 替身术
- Dead 死亡

关键逻辑：
1. 使用 enum 定义状态。
2. 玩家按下替身术键时，如果当前处于受击窗口，且查克拉足够，则消耗查克拉并进入 0.5 秒无敌。
3. 替身术触发时，在原地生成木桩替身对象，并把玩家移动到攻击者背后。
4. 长按 C 进入 Charge 状态，增加查克拉恢复速度，但移动速度降低或无法移动。
5. 攻击、施法、冲刺需要有前摇、持续、后摇三个阶段。

代码规范：
请写出 PlayerStateMachine 类，包含：
- currentState
- previousState
- transitionTo()
- canTransition()
- update(dt)
- onHit(attacker)
- trySubstitution(attacker)

请给出可直接放进浏览器游戏项目里的 TypeScript 代码。
```

### 提示词 4：忍术技能系统

```text
任务：
请设计一个通用的 JutsuSkill 忍术技能基类。

属性：
- id
- name
- chakraCost
- cooldown
- damage
- hitbox
- castTime
- recoveryTime
- elementType
- currentCooldown

方法：
- canCast(player)
- cast(player, world)
- update(dt)
- onHit(target, world)
- spawnVfx(world)

具体实现：
基于 JutsuSkill 创建一个 FireWheelJutsu 子类。

技能效果：
1. 玩家朝面向方向发射一个圆形火球。
2. 火球飞行 3 秒或碰撞敌人/墙体后爆炸。
3. 爆炸造成范围伤害，并附带短暂硬直。
4. 火球飞行时生成火星粒子，爆炸时生成冲击波。

要求：
请使用 TypeScript 编写，结构适合 Canvas 或 Phaser 项目。
不要依赖外部素材，视觉特效用基础图形和粒子数据描述即可。
```

### 提示词 5：连招系统

```text
任务：
请为《忍影试炼》编写一个连击计数器逻辑。

逻辑：
1. 玩家在 0.5 秒内连续按下 Attack，combo +1。
2. 超过 0.5 秒未攻击，combo 重置。
3. combo = 3 时，第三击触发浮空。
4. combo = 5 时，触发角色专属终结技。
5. 岚的终结技是风刃圆舞，绯月的终结技是雷影突袭，铁心的终结技是岩裂重击。

要求：
请包含：
- ComboCounter 类
- registerAttack()
- update(dt)
- reset()
- getComboEffect()
- 与 PlayerAttackSystem 的调用示例

输出 TypeScript 代码。
```

- 断桥区域有横向风力，影响跳跃轨迹。
